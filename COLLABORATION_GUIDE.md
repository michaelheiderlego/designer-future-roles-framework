## 🎯 Two Ways to Collaborate

### Option 1: GitHub Collaborative Editing (Recommended for Team)

**Everyone's changes are saved to GitHub and visible to all team members in real-time!**

#### Setup (One-Time, 2 minutes):

1. **Create a GitHub Personal Access Token**
   - Go to: https://github.com/settings/tokens
   - Click **"Generate new token (classic)"**
   - Give it a name: "Design Skills Framework"
   - Select scope: **`repo`** (Full control of private repositories)
   - Click **"Generate token"**
   - **Copy the token** (you won't see it again!)

2. **Connect to the Framework**
   - Open: https://michaelheiderlego.github.io/designer-future-roles-framework/
   - Click **"Connect GitHub"** button (top right)
   - Paste your token
   - Click OK

3. **Start Collaborating!**
   - You'll see: "✓ Connected as [your-username]"
   - Any changes you make are automatically saved to GitHub
   - Refresh to see changes from colleagues

## 💡 Best Practices

### For GitHub Collaborative Mode:

✅ **DO:**
- Refresh the page before making major edits (to see latest changes)
- Write clear, descriptive titles
- Add context in descriptions
- Test links before adding them

❌ **DON'T:**
- Share your GitHub token with others (everyone gets their own)
- Edit at the exact same time as a colleague (may cause conflicts)
- Delete items added by others without discussing first

### For Workshops:

**Real-time Collaboration:**
1. Everyone authenticates with GitHub
2. Project on screen, but everyone can edit from their device
3. Refresh periodically to see all changes
4. Natural discussion as items appear

**OR Traditional:**
1. Project on screen, one person types
2. Team dictates items
3. Export at end and share JSON file

## 🔄 Syncing with Team

### If Using GitHub Mode:

**See latest changes:**

## 🆘 Troubleshooting

### "Could not connect to GitHub"
- Check your internet connection
- Verify your token is correct
- Try generating a new token

### "Error saving to GitHub"
- You might not have permission to the repository
- Contact Michael to verify access
- Fall back to Local Mode temporarily

### "My changes disappeared"
- In Local Mode: Changes only saved in your browser
- In GitHub Mode: Check if you're still authenticated
- Refresh the page to reload from GitHub

### Can't see colleague's changes
- Make sure you're both in GitHub Mode
- Refresh your browser
- Check you're on the same repository
